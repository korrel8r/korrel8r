A _domain_ represents one type of signal or resource.
See [Introduction](../../introduction/) for an introduction to domains, classes, queries, and rules.

Each domain page below documents:

| Section | Describes |
|---------|-----------|
| **Object** | The data structure returned by queries. Usually a `struct` or `map` type. |
| **Store** | Configuration fields for connecting to the back-end service. |
| **Class** | List of classes for domains with a fixed class list. Description of class name format for domains with a variable set of classes |
| **Query** | Query syntax for the `selector` part of domain queries in `domain:class:selector` format. |
